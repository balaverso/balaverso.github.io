API de criptomoedas

API (Application Programming Interface) – É uma interface entre dois programas diferentes que se comunicam. Com ela terceiros disponibilizam uma interface de modo que possamos consumir um determinado serviço sem nos preocuparmos com a implementação. Web Service é uma API projetada para se comunicar obrigatoriamente via rede. Ou seja, quando uma API precisa enviar dados através da rede, podemos chamar de Web Services.

CRUD

As quatro operações básicas de armazenamento formam o CRUD

Criar Moeda

Exemplo de Requisição para criar uma nova moeda: /moedas/{moeda_id}

POST

/moedas/matic

Resposta

Primeiro, a mensagem de sucesso ao criar pela primeira vez. Ao tentar criar novamente uma moeda com id “matic”, o sistema emite uma mensagem de erro, informando que o id já existe.

Consultar Moeda

Requisição para listar todos as moedas do sistema, podendo opcionalmente receber filtros personalizados via path, de forma que se o cliente não definir nenhum parâmetro de consulta (nenhum filtro), os parâmetros receberão os valores padrão.

GET

/moedas

Requisição para visualizar os dados de uma moeda específica. Faz-se um GET de /moedas/{moeda_id}

GET

/moedas/btc

Resposta

Obtém-se um JSON com os dados da moeda requisitada.

Atualizar Moeda

Exemplo de Requisição para atualizar um novo moeda: /moedas/{moeda_id}

PUT

/moedas/sand

Resposta

O PUT também consegue criar uma nova moeda, emitindo uma mensagem de 201 Created a primeira vez, e 200 OK a partir da segunda requisição de update.

Deletar Moeda

Exemplo de Requisição para deletar uma moeda: /moedas/{moeda_id}

DELETE

/moedas/eth

Resposta

Primeiro, a mensagem de sucesso ao deletar uma moeda existente. Depois, ao tentar deletar a mesma moeda, obtém-se o erro 404, informando que a moeda não existe. .
